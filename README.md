# Skilss Assistant Bot

Agentic RAG Telegram-бот для навигации по IT-ролям и навыкам.

Бот: @skills_assistant_bot

## Проблема

Карьерная информация о ролях, навыках и компетенциях в компании распределена по разным источникам. Сотрудникам сложно понять, какие требования предъявляются к интересующей роли, чем отличаются смежные направления и что развивать для роста. HR тратит время на одинаковые вопросы.

## Решение

Telegram-бот, который отвечает на вопросы о ролях и навыках на основе внутренней базы знаний. Ответы строго из базы, не из общих знаний модели.

## Архитектура

Бот построен на Agentic RAG — подходе, при котором LLM выступает агентом и сама решает, как обработать запрос.

В отличие от линейного RAG (запрос → один поиск → ответ), агент может:
- делать несколько поисков по базе с разными формулировками
- задавать уточняющие вопросы пользователю
- отказываться отвечать на офтопик
- честно сообщать, если роли нет в базе

Компоненты:
- Эмбеддер: intfloat/multilingual-e5-base (768d, мультиязычный)
- Векторное хранилище: Qdrant Cloud
- LLM: OpenAI gpt-4o-mini 
- Улучшение поиска: Query Expansion через LLM
- Интерфейс: Telegram (pyTelegramBotAPI)

## Примеры работы

| Тип запроса | Пример | Поведение агента |
|---|---|---|
| Конкретный | "Навыки DevOps-инженера" | 1 поиск → ответ |
| Сравнительный | "Чем отличается PM от PjM" | 2 поиска → сравнение |
| Общий | "Как мне развиваться?" | Уточняющий вопрос |
| Офтопик | "Какая погода?" | Отказ без поиска |
| Нет в базе | "Навыки blockchain-разработчика" | Поиск →  ответ |

## Метрики качества

Оценка через LLM-as-a-Judge:

| Метрика | Значение | Цель | Что измеряет |
|---|---|---|---|
| Context Precision | 0.67 | >= 0.8 | Доля релевантных чанков в выдаче |
| Context Recall | 0.85 | >= 0.75 | Покрытие эталонного ответа контекстом |
| Faithfulness | 0.81 | >= 0.8 | Доля утверждений в ответе, подтвержденных контекстом |

Аудит агентской логики: 10/10 тестов по 5 категориям (single_search, multi_search, clarification, offtopic, hallucination_probe).
